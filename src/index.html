Claude Code Hooksモードについて

🔧 Claude Code Hooksモードとは

📡 グローバル監視機能: このアプリ以外のターミナルやエディターでClaude Codeを使用した際も、音声読み上げとVRM表情連動が働きます。
🎭 高度な感情分析: より詳細な感情分析により、キャラクターの表情がより自然で豊かに変化します。
🌐 システム全体対応: どこでClaude Codeを起動しても、このアプリが連動して動作します。

📱 アプリ内のみモード vs 🔧 Hooksモード

📱 アプリ内のみ（OFF）:
  • このアプリのターミナルでのみ動作
  • 他のターミナルには影響しない
  • より安定した動作
  • リソース使用量が少ない
🔧 Hooksモード（ON）:
  • システム全体のClaude Code活動を監視
  • 外部ターミナルでも音声・表情連動
  • より豊かな感情表現
  • 複数ターミナルで同時音声の可能性

💡 使い分けの目安

🎯 アプリ内のみがおすすめ:
  • このアプリだけでClaude Codeを使用
  • 安定した動作を重視
  • 他のアプリケーションに影響を与えたくない
⚡ Hooksモードがおすすめ:
  • VSCode、Vim、別ターミナルでもClaude Codeを使用
  • どこでも音声・表情連動が欲しい
  • より豊かなキャラクター表現を楽しみたい

⚠️ 注意事項

🔄 同時実行: 複数のターミナルで同時にClaude Codeを使用すると、音声が重複する場合があります。
💻 リソース: Hooksモードは常時監視のため、わずかにCPU使用量が増加します。
🔧 設定切り替え: モード切り替えは設定画面からいつでも変更可能です。

CLAUDE.md設定について

📝 CLAUDE.md設定とは

🤖 プロジェクト設定ファイル: AIアシスタント（モネ）のキャラクター設定やプロジェクト固有の指示を含むファイルを自動生成します。
🎯 一貫した対話: プロジェクトや作業内容に応じて、AIが適切な文脈で応答できるようになります。
📋 自動管理: AI起動時に作成され、停止時に自動削除されるため、手動管理が不要です。

⚙️ 自動生成 vs 手動生成

🔄 自動生成（ON）:
  • AI起動時に自動でCLAUDE.mdとAI.mdを生成
  • 常に最新の設定が反映される
  • 手間いらずで一貫した対話を実現
✋ 自動生成（OFF）:
  • 手動で必要な時のみファイルを生成
  • カスタマイズしたCLAUDE.mdを保持できる
  • 上級ユーザー向けの運用方法
🖱️ 手動生成ボタン:
  • 設定変更後にファイルを即座に再生成
  • プロジェクト変更時の設定更新に便利
  • 自動生成がOFFでも利用可能

📂 生成されるファイル

📄 CLAUDE.md: キャラクター設定（モネ）とプロジェクト共通の指示が含まれます。
📄 AI.md: 現在の作業ディレクトリに固有の情報やプロジェクト詳細が含まれます。
🗑️ 自動削除: AI停止時に両ファイルとも自動的に削除されるため、作業ディレクトリが汚れません。

💡 おすすめの使い方

✅ 初心者: 自動生成をONにして、手間なく一貫した対話を楽しむ。
⚡ 上級者: 自動生成をOFFにして、カスタマイズしたCLAUDE.mdで細かい設定を管理。
🔧 設定変更時: 作業ディレクトリやキャラクター設定を変更した後は手動生成で即座に反映。
